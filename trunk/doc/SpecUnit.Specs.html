SpecUnit

Console, when given a valid assembly file path

  • should write the spec report for the assembly indicated by the file path

Console, when no arguments are provided

  • should display an error message

Console, when there is no assembly that corresponds to the assembly name provided as a console argument

  • should display an error message

Context, when a context class has test methods

  • should provide the test methods as specifications

Context, when assigning a test concern to a test class

  • should indicate that it has a concern
  • should have a concern name that is the name specified by the context's concern attribute
  • the context name should be prefixed with the concern

Context, when creating a context for a type that has underscores in its name

  • should remove the underscores

Context, when creating a context for a type that is not a test fixture

  • should fail

Context, when not assigning a test concern to a test class

  • should indicate that it does not have a concern
  • should not have a concern name
  • should not have a context name that includes a concern

Context, when two contexts represent the same test fixture

  • should be equal

ReportGenerator, when rendering a context

  • should render the name as an H4
  • should render specifications as a bulletted list

ReportGenerator, when writing the spec report for an assembly

  • should write an html file named after the assembly that the report is created for

Specification, when a method has a test fixture attribute

  • should be a test method

Specification, when a specification name contains "if"

  • should add a warning to the specification name indicating that the specification may be a context

Specification, when a specification name contains "when"

  • should add a warning to the specification name indicating that the specification may be a context

SpecificationDataset, when creating a dataset for an assembly

  • should be named for the file stem of the name of the assembly

SpecificationDataset, when selecting context classes from a list of types

  • should ignore abstract types
  • should include concrete classes with the TestFixture attribute

SpecificationDataset, when selecting context classes from a list of types where the types are not from the same assembly

  • should fail

SpecificationDataset, when selecting context classes from an assembly

  • should include concrete classes with the TestFixture attribute

SpecificationDataset, when sorting contexts by context name

  • should place the first context at the beginning of the list of contexts
  • should place the last context at the end of the list of contexts

  • should allow "ShouldBeOfType" to be used in place of "Assert IsInstanceOfType"
  • should allow "ShouldNotBeOfType" to be used in place of "Assert IsNotInstanceOfType"
  • should allow "ShouldContain" to be used in place of "Assert Contains"
  • should allow "ShouldNotContain" to be used in place of "Assert NotContains"
  • should allow "ShouldBeEmpty" to be used on a collection in place of "Assert IsEmpty"
  • should allow "ShouldBeEmpty" to be used on a string in place of "Assert IsEmpty"
  • should allow "ShouldNotBeEmpty" to be used on a string in place of "Assert IsNotEmpty"
  • should allow "ShouldNotBeEmpty" to be used on a collection in place of "Assert IsNotEmpty"
